# 论文生成测试项目

## 项目概述

本项目用于测试和对比两个论���生成系统的效果：
1. **Paper Factory** — 当前项目，基于 Skill + Agent 混合架构
2. **Smart Query** — Smart Query OLD 目录下的智能问答系统（仅作参考，本次测试不使用）

## 论文主题

**标题**：大语言模型与领域知识图谱融合的智能数控故障诊断系统研究

**摘要**：本研究提出了一种智能数控故障诊断系统，将大语言模型与知识图谱相结合，旨在解决故障诊断知识的组织效率低、静态知识框架与动态工程环境之间缺乏适应性以及专家知识与实时数据流的整合困难等问题。

## 创新点

1. **知识表示与推理**：将数控故障诊断知识表示为结构化本体，支持复杂推理
2. **多模态融合**：结合语言模型的语义理解和知识图谱的符号推理
3. **动态适应性**：系统根据实时故障数据动态调整诊断策略
4. **专家知识整合**：将领域专家的隐性知识与实时数据流结合

## 目标会议/期刊

**推荐**：AAAI 2025、IJCAI 2025、ISWC 2025、WWW 2025
**关键词**：大语言模型（LLM）、知识图谱（Knowledge Graph）、数控系统（CNC）、故障诊断（Fault Diagnosis）、本体工程（Ontology）、语义推理（Semantic Reasoning）

## 系统要求

1. **使用当前 Paper Factory 的 config.json 配置**
2. **保持相同的 8.0 分评分标准**
3. **运行完整的 4-Phase pipeline**
4. **启用所有 5 个评审专家（包括 Domain Expert）**
5. **记录详细的迭代过程和 Token 消耗**

## 评估重点

对比两个系统在以下维度上的表现：
- **论文质量**：学术严谨性、逻辑连贯性、创新性
- **生成效率**：总耗时、迭代轮次
- **架构复杂度**：当前文件系统 vs 理论 Agent Teams
- **Token 消耗**：每轮迭代的平均消耗
